## Overview

The `DocsSidebar` component renders a sticky navigation sidebar based on your Fumadocs page tree. It supports top-level section links, auto-highlights the current page, filters out index pages, and includes gradient fades at the top and bottom.

## Import

```tsx
import { DocsSidebar } from '@olwiba/docs';
```

## Usage

```tsx
import { DocsSidebar, type SidebarSection } from '@olwiba/docs';
import { SidebarProvider } from '@olwiba/cn';

const sections: SidebarSection[] = [
  { name: 'Get Started', href: '/docs' },
  { name: 'Components', href: '/docs/components' },
  { name: 'Themes', href: '/docs/themes' },
];

function DocsPage({ pageTree }) {
  return (
    <SidebarProvider>
      <DocsSidebar tree={pageTree} sections={sections} />
      {/* Page content */}
    </SidebarProvider>
  );
}
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `tree` | `PageTree.Root` | The Fumadocs page tree structure |
| `sections` | `SidebarSection[]` | Top-level navigation sections shown above the page tree |
| `...props` | `SidebarProps` | All other props passed to the underlying Sidebar component |

## Features

- Sticky positioning below the header
- ScrollArea with gradient fades at top and bottom
- Filters out `index.mdx` pages from the navigation tree
- Section links with active state highlighting
- 220px width on mobile, 240px on desktop

## Mobile Navigation

On mobile viewports, the sidebar is hidden. Use [DocsMobileNav](/docs/components/docs-mobile-nav) to provide mobile navigation via a slide-out sheet.
